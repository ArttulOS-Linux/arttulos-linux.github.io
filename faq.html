01. THE_CONVERGENCE

What is the biological child of five ideologies? [ + ]

ArttulOS is the biological child of AcreetionOS, Arch, Nix, RHEL, and Rocky Linux. It inherits Acreetion's brutalist transparency, RHEL/Rocky's immutable stability, Arch's multi-threaded velocity, and Nix's mathematical reproducibility. It is not a wrapper; it is a fundamental reconfiguration of the Linux ecosystem.

How is Natalie Spiva involved? [ + ]

ArttulOS is brought to you by the Co-Lead of AcreetionOS, Natalie Spiva. The project maintains the same uncompromising brutalist values and security-first design philosophy of Acreetion while introducing a high-density, multi-layer engine for enterprise-desktop convergence.

What is the PacNixum Engine? [ + ]

PacNixum is our custom-built orchestration layer. It allows the system to resolve dependencies across three distinct formats simultaneously:

  • RPM: For RHEL-native binary stability.
  • Pacman: For Arch-rolling speed and AUR access.
  • Nix: For declarative, isolated system components.

02. PROVISIONING_&_LOGIC

How do I manage the hybrid layers? [ + ]

The pacnixum command is the primary interface. It detects the best source layer automatically or allows you to specify one.

# Automatic resolution
pacnixum install firefox

# Force a specific layer
pacnixum install --layer=rhel nginx
pacnixum install --layer=aur spotify
How do Generations and Rollbacks work? [ + ]

Borrowing from NixOS logic, every system change creates a new Generation. If a configuration change or update breaks your environment, you can select any previous generation at the bootloader (GRUB/Systemd-boot) to instantly revert the entire system state. Rollbacks are a human right.

03. SOVEREIGN_SECURITY

Does ArttulOS support SELinux? [ + ]

Yes. ArttulOS ships with SELinux in Enforcing Mode by default, maintaining RHEL/Rocky security standards. We also provide "Acreetion-Hardened" kernel profiles that reduce the attack surface by disabling unnecessary kernel modules at the source.
