RHEL One-to-One Conversion — Work in Progress

Short summary: ArttulOS is beginning a conversion to become a RHEL One-to-One compatible distribution for consumer use. This is a technical and legal work-in-progress — do not consider any downloads or images production-ready.

What "One-to-One compatible" means

A one-to-one compatible rebuild aims to reproduce the functionality and binary compatibility of Red Hat Enterprise Linux (RHEL) by rebuilding the publicly-available RHEL sources while avoiding Red Hat trademarks and proprietary artwork. The goal is to provide a stable, downstream-compatible distribution for users and organizations that want a RHEL-like experience without vendor lock-in.

Project goals

  • Provide binary compatibility with RHEL ABI where practical.
  • Maintain a transparent build and release process using reproducible rebuilds from vendor-provided source RPMs.
  • Offer consumer-focused images and an easy upgrade path for desktop users.
  • Respect trademarks and avoid the use of Red Hat's logos, branding, and trademarked names.

Current status (WIP)

  • Initial planning and repository layout: complete.
  • Build automation prototypes: in progress.
  • Package rebuild pipeline (SRPM ingestion): in development.
  • Consumer desktop images (ISO/installer): not ready — DO NOT USE FOR PRODUCTION.

Roadmap

  1. Establish legal/trademark guidelines and documentation (completed — see TRADEMARKS.md).
  2. Set up rebuild infrastructure and CI for SRPM builds.
  3. Produce minimal server images for testing and validation.
  4. Create consumer desktop images and installer tooling.
  5. Public beta and community testing.

Legal & Trademark note

We are not affiliated with Red Hat, Inc. and will not use Red Hat trademarks, logos, or artwork. We will follow best-effort guidance to avoid trademark infringement; for details see TRADEMARKS.md.

This effort is led by Natalie Spiva — co-founder of ArttulOS Linux — and the ArttulOS team. The project is community-driven and welcomes contributors.
